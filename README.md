# JSAI OSCE JP 2026 用データおよびプロンプト

本リポジトリは，論文で使用したデータおよびプロンプトを公開するものです．
研究の再現性確保および理解促進を目的としています．

---

## ディレクトリ構成

### 1. prompt/

各種エージェントおよび評価用LLMに与えるシステムプロンプトを格納しています．

#### prompt_for_doctor/

データ合成時に使用する，医者役エージェントに与えるシステムプロンプトです．

#### prompt_for_patient/

仮想模擬患者に関するプロンプト群です．用途ごとに分かれています．

* **QC_aware_answering.txt**
  初期回答生成時に使用するシステムプロンプト

* **judge_in_self-refine.txt**
  Self-refine において，回答の評価およびフィードバック生成時に使用するシステムプロンプト

* **post_feedback_ans_in_self-refine.txt**
  フィードバックを考慮した再回答生成時に使用するシステムプロンプト

#### judge_llm/

評価用LLMに与えるシステムプロンプトです．
評価観点ごとに個別のプロンプトが用意されています．

---

### 2. data_for_test/

VSPの有効性評価実験に使用するテストデータです．

* **question_list/**

### 3. data_for_train/

合成データ生成および学習に使用するデータです．

* **scenario_data/**
  合成データ生成時に使用するシナリオデータ（50件）

* **dataset.json**
  対話形式の学習データを格納したJSONファイルです．

---

## dataset.json の構造

`dataset.json` は，シナリオごとに複数の対話を含む階層構造となっています．

各シナリオは以下の形式で構成されます：

```json
"scenario_X": {
    "dialogue_1": {
        "user_input": "...",
        "chosen": "...",
        "rejected": "..."
    },
    ...
}
```

### 各フィールドの説明

* **scenario_X**
  シナリオ単位のデータ（Xは番号）

* **dialogue_Y**
  対話ターン（Yは発話順）

* **user_input**
  医者側の発話（入力）

* **chosen**
  採用された患者側の応答

* **rejected（任意）**
  不採用の応答（存在しない場合もある）

---

## 学習用途について

* **rejected が存在しない場合**
  → SFTのみで使用

* **rejected が存在する場合**
  → SFTに加え，DPOに利用可能
